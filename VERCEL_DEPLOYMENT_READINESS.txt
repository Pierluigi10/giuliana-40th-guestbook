                       QUICK START - 3 DEPLOYMENT OPTIONS
================================================================================

OPTION 1: AUTOMATED SCRIPT (Recommended)
  $ ./scripts/deploy-to-vercel.sh        # Preview deployment
  $ ./scripts/deploy-to-vercel.sh --prod # Production deployment

  What it does:
  - Checks Node.js and npm versions
  - Verifies git working tree clean
  - Runs npm build
  - Runs TypeScript type check
  - Runs linting
  - Verifies environment variables
  - Deploys using Vercel CLI

OPTION 2: VERCEL DASHBOARD (Easiest for First Time)
  1. Go to https://vercel.com/dashboard
  2. Click "Add New" → "Project"
  3. Select "g_gift" repository
  4. Add environment variables in Settings tab
  5. Click "Import"
  6. Vercel auto-deploys

OPTION 3: VERCEL CLI
  $ npm install -g vercel              # Install if needed
  $ vercel                             # Preview deployment
  $ vercel --prod                      # Production deployment

================================================================================
                           BRANCH STRATEGY
================================================================================

PRODUCTION:
  main → Vercel auto-deploys when you push
  └─ https://guestbook-app.vercel.app

DEVELOPMENT/PREVIEW:
  dev → Create feature branches from here
  ├─ feat/feature-name → Creates preview URL automatically
  └─ Merge to dev when ready to test

FEATURE BRANCH WORKFLOW:
  1. Create branch from dev: git checkout -b feat/my-feature
  2. Push to GitHub: git push -u origin feat/my-feature
  3. Create PR to dev (Vercel creates preview URL)
  4. Test on preview URL
  5. Merge PR to dev
  6. When ready, merge dev → main (production)
